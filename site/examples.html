Examples

Command Examples

Practical examples for all current commands. All examples assume bomi is installed and available on your PATH.

Basic search

bomi search "10k 0402 resistor"

bomi search "100nF capacitor" --basic-only

bomi search "LED" --preferred-only --min-stock 1000

bomi search "STM32" --pages 3 --limit 50

search queries the live JLCPCB catalog and caches results locally. Use --basic-only to restrict to JLCPCB basic parts (lower assembly cost). Use --preferred-only for JLCPCB preferred parts.

search for USB-C connector and LED driver

Attribute filtering

bomi search "0402 resistor" --attr "Resistance >= 10k"

bomi search "0402 capacitor" --attr "Voltage Rated >= 25"

bomi search "MOSFET SOT-23" \
  --attr "Drain Source Voltage (Vdss) >= 30" \
  --attr "Continuous Drain Current (Id) >= 5"

bomi search "LDO 3.3V" --max-price 0.50 --basic-only

Operators: >= <= > < = !=. Multiple --attr flags are ANDed. SI prefixes are supported: 10k, 100n, 4.7u, 1M.

Output formats

Most commands accept --format table|json|csv|markdown.

Search as JSON

bomi search "10k resistor" --format json
{
  "status": "ok",
  "command": "search",
  "count": 2,
  "results": [
    {
      "lcsc_code": "C8287",
      "mfr_part": "RC0402FR-0710KL",
      "manufacturer": "YAGEO",
      "package": "0402",
      "stock": 500000,
      "price_qty1": 0.0037
    }
  ]
}

BOM as JSON

bomi bom --format json
{
  "status": "ok",
  "command": "bom",
  "data": [
    {
      "ref": "R1",
      "lcsc": "C8287",
      "quantity": 2,
      "warnings": []
    }
  ]
}

Note: bom uses "data" instead of "results".

Local cache queries

bomi query

bomi query "resistor" --package 0402

bomi query --attr "Resistance >= 1k" --attr "Resistance <= 100k" --basic-only

query searches only the local cache — no network required. Useful for scripting and offline filtering.

Inspect and compare

bomi fetch C8287
bomi info C8287

bomi fetch C8287 C25900
bomi compare C8287 C25900

bomi fetch C8287 --force

fetch caches one or more parts. --force refreshes even if the part was recently cached. info shows full specs, attributes, and links. compare shows a side-by-side table.

Datasheets

bomi fetch C347356
bomi analyze C347356 \
  --prompt "Summarize power rating and operating temperature range"

bomi analyze C347356 --model "anthropic/claude-sonnet-4.6"

bomi datasheet C347356 --pdf -o docs/datasheets/

bomi datasheet C347356 --pdf --summary -o docs/datasheets/

analyze sends the datasheet PDF to OpenRouter and returns structured markdown. datasheet --summary writes a .md file alongside the PDF. Both require an OpenRouter API key.

The default prompt already asks for specs, pins, application values, and design notes. Running without --prompt gives useful output for most parts: 

bomi analyze C347356

Add --prompt only when you need something specific:

bomi analyze C347356 --prompt "What is the sense resistor formula for 350mA output?"
UMW PT4115  C347356  LED Constant Current Driver Processed from PDF datasheet using anthropic/claude-sonnet-4.6

Project workflow

Start a new project

mkdir my-board && cd my-board
git init
bomi init --name "my-board" --description "Motor driver board"

Research and select components

bomi search "100nF 0402 X7R"
bomi fetch C1525
bomi info C1525

bomi select C1525 --ref C1 --qty 1 --notes "Bypass cap"
bomi select C1525 --ref C2 --qty 1 --notes "Bypass cap"
bomi select C347356 --ref U2 --qty 1 --notes "LED driver"

BOM review

bomi bom
bomi bom --check
bomi bom --format csv
bomi bom --format markdown
bomi status
bom + status — cost rollup and stock summary

Refine and export

bomi relabel R1 R3
bomi deselect C2
bomi bom --check
bomi bom --format csv
bomi bom --format json
relabel, validate, and export BOM

Work from outside the project directory

bomi --project ~/Projects/my-board bom

export BOMI_PROJECT=~/Projects/my-board
bomi status

Commit the result

git add .bomi/project.yaml
git commit -m "Select initial BOM components"

Database commands

bomi db stats

bomi db clear

The shared cache lives at ~/Library/Application Support/bomi/parts.db on macOS or ~/.local/share/bomi/parts.db on Linux. db clear wipes all cached parts — they can be re-fetched at any time.
