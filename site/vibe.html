Getting started

Vibe circuit design

Set up a project, point your AI agent at the tool guide, then find and verify parts through conversation — search by spec, compare candidates, pull datasheets, build the BOM — all backed by real JLCPCB catalog data.

Prerequisites

Before you start

  • Install jlcpcb-tool: uv tool install jlcpcb-tool
  • A Claude Code, Cursor, Copilot, or other terminal-capable AI agent
  • Optional: an OpenRouter API key for datasheet analysis

Most commands work without an OpenRouter key — you only need it for jlcpcb analyze and jlcpcb datasheet --summary.

Setup

Four steps to get started

01

Create a project folder

Start with a normal directory and initialize git. The BOM will be committed here so part decisions are tracked alongside your design files. 

mkdir my-board && cd my-board
git init
02

Initialize jlcpcb project metadata

jlcpcb init creates .jlcpcb/project.yaml in the current directory. This file is the source of truth for your BOM — commit it with every change. It also appends PDF ignore rules to .gitignore. 

jlcpcb init --name "my-board" --description "What this board does"

You can verify the project was created: 

cat .jlcpcb/project.yaml
03

Launch your AI agent

Open Claude Code, Cursor, GitHub Copilot, or any agent that can run terminal commands from your project directory. The agent needs to be able to call jlcpcb directly.

Claude Code Cursor Agent GitHub Copilot Aider
04

Tell the agent to read the tool guide

Give the agent this prompt to load the tool guide. It covers all commands, output formats, caching rules, and project context — everything the agent needs to use jlcpcb correctly.

Paste this into your agent:

Read the jlcpcb-tool agent guide at this URL: https://somebox.github.io/jlcpcb-tool/guide.html Then use jlcpcb to help me find parts for my PCB project. We need [describe what you're building].

Alternatively, you can paste the raw markdown from docs/jlcpcb-tool-guide.md directly into the agent's context.

Working with your agent

What to ask

Once the agent has read the guide, it can handle the full part research and BOM workflow. Here are the kinds of tasks it can do — and example prompts to get started.

Find parts

Find a 3.3V buck converter in a small package, basic parts only
Search for a Schottky diode, SOD-123, rated for at least 40V and 1A
Find 100nF X7R capacitors in 0402 with at least 10V rating
Look for a tactile switch with 100gf actuating force, basic part if possible

Compare and decide

Compare C8287 and C25900 and recommend the better one for a 10k pull-up on a 3.3V GPIO
Is C347356 a good choice for driving a 1W LED from USB-C? Check the datasheet
Find two alternative MOSFETs to C20917 and compare them

Datasheets and analysis

Download the datasheet for C347356 and summarize the external component requirements
What is the sense resistor formula for the PT4115 at 350mA output current?
Save a markdown summary of the C456012 datasheet to docs/datasheets/

Build and manage the BOM

Add C347356 as U1 with notes "LED driver, PT4115"
Update the BOM: R1 = C8287 × 2, R2 = C8287 × 1, both are 10k pull-ups
Show me the current BOM with stock status
Export the BOM as markdown and save it to docs/bom.md
Remove D1 from the BOM and replace it with C4928

Cost and availability

Estimate the total BOM cost at 50 units and at 500 units
Flag any parts with stock under 1000 or that are not basic parts
Find alternatives for any out-of-stock components

Commit and review

Run bom --check to refresh stock and pricing, then commit the result
Show me what changed in project.yaml since the last commit

Tips

Getting the most out of agent-assisted design

Be specific about constraints

Mention package size, voltage rating, current, and whether you want basic parts. The more specific you are, the better the agent can filter with --attr.

Use the local cache

After searching, the agent can use query for fast offline filtering. This is useful when iterating on attribute ranges without re-hitting the catalog.

Track decisions in notes

When adding BOM entries, ask the agent to include --notes explaining why the part was chosen. This makes future BOM reviews much easier.

Commit regularly

The BOM is in .jlcpcb/project.yaml. Commit it after each meaningful change so you have a history of every part decision with context.
