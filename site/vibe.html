Getting started

Agent-assisted circuit design

Set up a project, point your AI agent at the tool guide, then work through component research conversationally — search by spec, compare candidates, pull datasheets, build and revise the BOM.

See it in action

Agent BOM review

The agent runs jlcpcb tool calls to check stock levels, finds a part that is out of stock, searches for a replacement with the same footprint, and proposes the swap — all from a single prompt.

See the examples page for individual command recordings.

review bom · check stock · find replacement
Prerequisites

Before you start

  • Python 3.11+ and uv installed
  • A Claude Code, Cursor, Copilot, or other terminal-capable AI agent
  • Optional: an OpenRouter API key for datasheet analysis

Install jlcpcb-tool

Clone the repo and install the jlcpcb command globally with uv:

git clone https://github.com/somebox/jlcpcb-tool
cd jlcpcb-tool
uv tool install -e .

Verify it works: jlcpcb --help

Setup

Enabling a Project

01

Choose a project folder

Start with a normal directory (or existing project folder) and make sure its tracked in git. The BOM will be committed here alongside your design files.

mkdir my-board && cd my-board
git init
02

Initialize project metadata

Creates .jlcpcb/project.yaml and appends datasheet PDF rules to .gitignore. This file is the BOM source of truth — commit it with every change. 

jlcpcb init --name "my-board" --description "What this board does"
03

Launch your AI agent

Open Claude Code, Cursor, Copilot, or any agent that can run terminal commands from your project directory.

Claude Code Cursor Agent GitHub Copilot Aider
04

Tell the agent to read the tool guide

The guide covers all commands, output formats, caching rules, and project context. Paste this to get started:

Read the jlcpcb-tool agent guide at:
https://somebox.github.io/jlcpcb-tool/guide.html
Then use jlcpcb to help me find parts for my PCB project. We need [describe what you're building].

Alternatively, paste the raw markdown from docs/jlcpcb-tool-guide.md directly into the agent's context.

Working with your agent

What to ask

Once the agent has read the guide, it can handle the full research and BOM workflow.

Find parts

Find a 3.3V buck converter in a small package, basic parts only
Search for a Schottky diode, SOD-123, rated for at least 40V and 1A
Find 100nF X7R capacitors in 0402 with at least 10V rating

Compare and decide

Compare C8287 and C25900, recommend the better one for a 10k pull-up on 3.3V GPIO
Is C347356 suitable for driving a 1W LED from USB-C? Check the datasheet

Datasheets

Download the C347356 datasheet and summarize external component requirements
Save a markdown summary of the C456012 datasheet to docs/datasheets/

Build and update the BOM

Add C347356 as U1 with notes "LED driver, PT4115"
Update the BOM: R1 = C8287 × 2, R2 = C8287 × 1, both 10k pull-ups
Export the BOM as markdown and save to docs/bom.md

Cost and availability

Estimate BOM cost at 50 and 500 units
Flag any parts with stock under 1000 or not marked as basic
Find alternatives for any out-of-stock components

Revisions and lifecycle

Run bom --check to refresh stock and pricing, then commit the result
Swap R1 from C8287 to C25900 and update the notes to explain why
Tips

Getting the most out of it

Be specific about constraints

Mention package size, voltage, current, and whether you want basic parts. More detail means better --attr filtering.

Use the local cache

After searching, use query for fast offline filtering. Useful when iterating on attribute ranges without re-hitting the catalog.

Notes in selections

Ask the agent to include --notes explaining why a part was chosen. Makes future BOM reviews much easier.

Commit regularly

.jlcpcb/project.yaml is plain text. Commit after each meaningful BOM change to keep a history of every part decision.
