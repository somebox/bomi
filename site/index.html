What it does

From catalog search to committed BOM

⌕

Part research

Live search across the JLCPCB catalog. Filter by package, stock, price, and attributes like resistance, voltage, or current. Results are cached locally for fast offline access. 

jlcpcb search "buck converter 3.3V"
jlcpcb search "0402 resistor" \
  --attr "Resistance >= 10k" \
  --basic-only
⊞

Inspect and compare

Fetch exact LCSC codes, view full specs, and compare parts side-by-side. All commands support --format json for scripting and agent use. 

jlcpcb fetch C347356 C456012
jlcpcb compare C347356 C145837
jlcpcb info C347356 --format json
≡

BOM as code

Selections live in .jlcpcb/project.yaml alongside your design files. BOM changes are diff-able, reviewable, and reproducible. 

jlcpcb init --name "my-board"
jlcpcb select C347356 --ref U1
jlcpcb bom --check
jlcpcb status
⬡

Datasheet analysis

Download datasheet PDFs and run AI analysis via OpenRouter. Get summaries, pin tables, ratings, and application notes as structured markdown. 

jlcpcb analyze C347356 \
  --prompt "Summarize pin functions"
jlcpcb datasheet C347356 \
  --pdf --summary -o docs/
In action

BOM review and cost estimate

jlcpcb bom produces a full table with stock status and per-unit pricing. jlcpcb status gives an overview with warnings, quantity rollup, and rough cost.

Run bom --check before ordering to refresh stock and pricing from the live catalog. Export as CSV, JSON, or markdown for downstream use.

The BOM file is plain YAML, committed with your design. Every change is tracked in git so you can see exactly when a part was swapped, why, and by whom.

Agentic workflows

Built for AI-assisted design

Every command produces --format json output with a consistent envelope — status, command name, count, results. Agents can search, inspect, compare, and update the BOM without opening a browser.

The agent guide is a short, structured reference designed to be read at the start of a Claude Code or Copilot session. It gives the agent everything it needs to work with the tool: command syntax, output shapes, and project context rules.

Works with any AI assistant that can execute terminal commands:

Claude Code GitHub Copilot Cursor Agent Aider any terminal-capable agent

Example agent prompts

Find a 3.3V LDO in SOT-23, basic parts only, at least 500mA
Compare the two resistor options and recommend the better one for 3.3V GPIO pull-up
Download the PT4115 datasheet and summarize the switching frequency and sense resistor formula
Update the BOM with U1 = C347356, R1 = C8287 × 2, add notes for each
Estimate the BOM cost at 100 units and flag anything with stock under 1000
Search for an alternative to C347356 if it goes out of stock
Export the BOM as markdown and add it to docs/bom.md
Getting started

Vibe circuit design

Set up a project in a few commands, point Claude (or your agent of choice) at the tool guide, and start finding parts conversationally. Search by spec, compare candidates, pull datasheets, build the BOM — all through natural language backed by real catalog data.

Getting started guide → 
mkdir my-board && cd my-board
git init
jlcpcb init --name "my-board"

#  [URL]. Find me a 3.3V buck
#  converter, basic parts only,
#  SOT-23 or smaller."
Install and configure

Install

uv tool install jlcpcb-tool

Or install in editable mode for development: 

git clone https://github.com/somebox/jlcpcb-tool
cd jlcpcb-tool
uv sync
uv tool install -e .

Configure

Most commands work without any configuration. Add an OpenRouter key only if you want datasheet analysis: 

# macOS: ~/Library/Application Support/jlcpcb/config.yaml
# Linux: ~/.local/share/jlcpcb/config.yaml

openrouter_api_key: sk-or-v1-...

Or set JLCPCB_OPENROUTER_API_KEY in your environment.
